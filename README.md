## 📁 Folder Structure

Every solution is stored in its own folder, named by problem number and title for easy navigation:

```
📂 leetcode-solutions/
 ┣ 📂 0001-two-sum/
 ┃ ┗ solution.py
 ┣ 📂 0021-merge-two-sorted-lists/
 ┃ ┗ solution.py
 ┣ 📂 0042-trapping-rain-water/
 ┃ ┗ solution.cpp
 ┣ 📂 0121-best-time-to-buy-and-sell-stock/
 ┃ ┗ solution.js
 ┗ ...
```

- Each folder = one problem
- Folder name format → `{4-digit-number}-{problem-title}`
- File name → `solution.{language extension}`

---

## 🔁 What Happens If I Resubmit a Problem?

If you solve the same problem again (better approach, different language, optimized solution) the extension **overwrites** the existing file in the folder with your latest submission.

This means:
- ✅ Your repo always has your **most recent** solution
- 🔄 Old solutions are replaced, not duplicated
- 💡 If you want to keep both — rename the old file to `solution_v1.py` before resubmitting
